## 二、待修复问题

### 问题1：扣分可扣到负分

**当前问题**：扣分时检查积分是否充足，不允许扣成负分

**修复方案**：移除扣分时的积分检查，允许扣到负分；保留兑换的积分检查

---

### 问题2：历史记录新建表

**当前问题**：历史记录存储在 kids 表的 history JSON 字段中

**修复方案**：
1. 新建 `point_history` 表
2. 修改添加/扣分/兑换逻辑，写入新表
3. 修改查看历史记录，读取新表

**新表结构**：
```sql
CREATE TABLE point_history (
    id SERIAL PRIMARY KEY,
    kid_id INTEGER REFERENCES kids(id),
    type VARCHAR(20),  -- 'add'/'subtract'/'exchange'
    points INTEGER,
    reason VARCHAR(255),
    parent_name VARCHAR(100),
    created_at TIMESTAMP DEFAULT NOW()
);
```

---

### 问题3：多家长关联 + 创建者删除

**当前问题**：
- parent_id 是单个值，关联会覆盖
- 任何家长都可以删除小朋友

**修复方案**：
1. 修改 kids 表结构：
   - 新增 `creator_id` 字段（创建者昵称）
   - `parent_id` 改为 `parents`（数组，存储多个家长）
2. 修改添加逻辑：设置 creator_id
3. 修改关联逻辑：追加到 parents 数组
4. 修改删除逻辑：检查是否是 creator_id

### 表1：kids（修改）

```sql

ALTER TABLE kids ADD COLUMN creator_id VARCHAR(100);

-- 将现有的 parent_id 改为支持多家长（如果需要迁移数据）
-- 如果使用 JSONB 类型
ALTER TABLE kids ADD COLUMN parents JSONB;
```

---

### 表2：point_history（新建）

```sql
CREATE TABLE point_history (
    id SERIAL PRIMARY KEY,
    kid_id INTEGER REFERENCES kids(id) ON DELETE CASCADE,
    type VARCHAR(20) NOT NULL,  -- 'add'/'subtract'/'exchange'
    points INTEGER NOT NULL,
    reason VARCHAR(255),
    parent_name VARCHAR(100),
    created_at TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_point_history_kid_id ON point_history(kid_id);
CREATE INDEX idx_point_history_created_at ON point_history(created_at DESC);
```
