【功能清单】（2026-03-19 更新）
================================================================================

基础功能：
1. 用户登录 - 使用昵称登录，数据存储在 localStorage
2. 添加小朋友 - 填写姓名和初始积分，支持自动生成6位关联码
3. 关联小朋友 - 通过6位关联码关联其他家长添加的小朋友
4. 积分管理 - 添加积分、扣除积分、积分兑换
5. 积分排名 - 显示所有小朋友的积分排名（含缓存优化）
6. 历史记录 - 查看每个小朋友的积分变动历史
7. 删除小朋友 - 删除小朋友及其相关记录

新增功能（2026-03-14）：
8. 扣分可到负分 - 扣分不再检查积分是否充足，允许扣到负数
9. 兑换检查积分 - 兑换时仍会检查积分是否充足
10. 多家长关联 - 支持多个家长关联同一个小朋友（用逗号分隔）
11. 创建者权限 - 只有创建者可以删除小朋友
12. 历史记录表 - 使用独立的 point_history 表存储历史记录

新增功能（2026-03-19）：
13. 自动登录 - 页面加载时自动检查缓存的昵称，实现自动登录
14. 昵称预填充 - 退出登录后，输入框自动填充上次使用的昵称
15. 历史记录RLS修复 - 添加 point_history 表的 RLS 策略，解决前端无法查询历史记录的问题

【数据库变更SQL】
-- 添加 creator_id 字段
ALTER TABLE kids ADD COLUMN creator_id VARCHAR(100);

-- 创建历史记录表
CREATE TABLE point_history (
    id SERIAL PRIMARY KEY,
    kid_id INTEGER REFERENCES kids(id) ON DELETE CASCADE,
    type VARCHAR(20) NOT NULL,
    points INTEGER NOT NULL,
    reason VARCHAR(255),
    parent_name VARCHAR(100),
    created_at TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_point_history_kid_id ON point_history(kid_id);
CREATE INDEX idx_point_history_created_at ON point_history(created_at DESC);

【开发注意事项】
================================================================================

1. Android 兼容性问题：登录后主界面可能不显示，需要在 showMain() 函数中添加：
   - loginSection.style.visibility = 'hidden';
   - mainSection.style.visibility = 'visible';
   - mainSection.style.opacity = '1';
   - mainSection.offsetHeight; // 触发重绘
   - document.body.style.background = 'linear-gradient(135deg, #667eea 0%, #764ba2 100%)';

2. 积分排名缓存：30秒缓存，避免频繁查询数据库

3. 字段名注意：数据库使用 total_points（蛇形命名），代码中使用 kid.total_points

4. 生产发布流程：
   - 修改 dev/index.html 进行测试
   - 测试通过后，复制 dev/index.html 到 index.html
   - 同时提交两个文件到 main 分支

5. parent_id 存储格式：多家长用逗号分隔，如 "家长A,家长B,家长C"

6. 查询逻辑：
   - 我的小朋友：查询 parent_id 包含当前用户 OR creator_id 是当前用户
   - 积分排名：查询 parent_id 包含当前用户 OR creator_id 是当前用户
